1. SUPABASE — CONFIGURAR PROJETO
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1.1 Criar projeto em https://app.supabase.com
    → Guardar: URL do projeto e anon key (Settings → API)

1.2 Correr as migrations SQL pela ordem abaixo
    (Supabase Dashboard → SQL Editor → New Query)

    FICHEIRO 1: supabase/migrations/001_schema.sql
    → Cria as tabelas: profiles, coach_usage
    → Cria o trigger automático de perfil ao registar
    → Activa Row Level Security (RLS)

    FICHEIRO 2: aplicar ESTE patch ANTES de correr o ficheiro 2
    (corrige o check constraint do plano para incluir 'annual'):

    ┌──────────────────────────────────────────────────────────────────────┐
    │  alter table public.profiles                                          │
    │    drop constraint if exists profiles_plan_check;                     │
    │                                                                       │
    │  alter table public.profiles                                          │
    │    add constraint profiles_plan_check                                 │
    │    check (plan in ('free', 'pro', 'annual'));                         │
    └──────────────────────────────────────────────────────────────────────┘

    FICHEIRO 3: supabase/migrations/002_community.sql
    → Cria as tabelas: community_posts, community_comments, community_likes

1.3 Activar confirmação de email (opcional mas recomendado)
    → Authentication → Email → Enable email confirmations: ON

1.4 Configurar URL do site
    → Authentication → URL Configuration
    → Site URL: https://jobjump.co
    → Redirect URLs: https://jobjump.co/**

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
2. STRIPE — CONFIGURAR PAGAMENTOS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

2.1 Criar conta em https://dashboard.stripe.com (se ainda não tiver)

2.2 Criar produto "JobJump Pro"
    → Products → Add product
    → Nome: JobJump Pro
    → Criar 2 preços para este produto:

    PREÇO 1 — Pro Mensal
    → Tipo: Recurring
    → Valor: R$ 29,90
    → Intervalo: Monthly
    → Guardar o Price ID (começa com "price_...") → será STRIPE_PRO_PRICE_ID

    PREÇO 2 — Pro Anual
    → Tipo: Recurring
    → Valor: R$ 238,80
    → Intervalo: Yearly
    → Guardar o Price ID → será STRIPE_ANNUAL_PRICE_ID

2.3 Configurar o Portal do Cliente (para o utilizador cancelar/gerir)
    → Settings → Billing → Customer portal → Activate
    → Copiar o link do portal e atualizar em:
       src/components/dashboard/SettingsPage.jsx
       (linha com "https://billing.stripe.com/p/login/test_xxx")

2.4 Configurar Webhook
    → Developers → Webhooks → Add endpoint
    → URL: https://<ID>.supabase.co/functions/v1/stripe-webhook
      (substituir <ID> pelo ID do projeto Supabase, visível no URL do dashboard)
    → Eventos a escutar:
       • customer.subscription.created
       • customer.subscription.updated
       • customer.subscription.deleted
    → Guardar o Webhook Signing Secret → será STRIPE_WEBHOOK_SECRET

2.5 Copiar chaves de API
    → Developers → API keys
    → Publishable key (pk_live_...) → será VITE_STRIPE_PUBLISHABLE_KEY
    → Secret key (sk_live_...) → será STRIPE_SECRET_KEY

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
3. ANTHROPIC — CHAVE DE API (Coach IA)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

3.1 Criar conta em https://console.anthropic.com
3.2 API Keys → Create Key
    → Guardar → será ANTHROPIC_API_KEY

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
4. VARIÁVEIS DE AMBIENTE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

4.1 Criar ficheiro .env.local na raiz do projeto (NÃO commitar no git):

    VITE_SUPABASE_URL=https://xxxxxxxxxxxxxxxxxxx.supabase.co
    VITE_SUPABASE_ANON_KEY=eyJ...
    VITE_STRIPE_PUBLISHABLE_KEY=pk_live_...

4.2 Configurar segredos nas Edge Functions do Supabase
    (via Supabase CLI ou Dashboard → Edge Functions → Secrets)

    Usando o CLI (na pasta do projeto):

    supabase secrets set ANTHROPIC_API_KEY=sk-ant-...
    supabase secrets set STRIPE_SECRET_KEY=sk_live_...
    supabase secrets set STRIPE_PRO_PRICE_ID=price_...
    supabase secrets set STRIPE_ANNUAL_PRICE_ID=price_...
    supabase secrets set STRIPE_WEBHOOK_SECRET=whsec_...

    Alternativa via Dashboard:
    → Project → Edge Functions → Manage secrets → Add secret

5. EDGE FUNCTIONS — DEPLOY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Instalar Supabase CLI (se ainda não tiver):
    npm install -g supabase

Ligar ao projeto:
    supabase login
    supabase link --project-ref <ID_DO_PROJETO>

Deploy de todas as funções:
    supabase functions deploy create-checkout
    supabase functions deploy stripe-webhook
    supabase functions deploy chat  (se existir — Coach IA)

Verificar se estão activas:
    → Supabase Dashboard → Edge Functions → deve aparecer como "Active"

6. DEPLOY DO FRONTEND
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Opção recomendada: Vercel ou Netlify (deploy automático ao fazer push)

VERCEL:
    1. https://vercel.com → Import Project → ligar repositório GitHub
    2. Framework: Vite
    3. Environment Variables: adicionar as VITE_* do passo 4.1
    4. Deploy

NETLIFY:
    1. https://netlify.com → Add new site → Import from Git
    2. Build command: npm run build
    3. Publish directory: dist
    4. Environment Variables: adicionar as VITE_* do passo 4.1
    5. Deploy

Após o deploy, atualizar a URL do site no Supabase (passo 1.4).
